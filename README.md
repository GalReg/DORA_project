# DORA project

## Workflow  
Мы используем **GitHub Flow**:  
1. `main` — стабильная ветка.  
2. Для задач создавайте ветки от `main`:  
   ```bash  
   git checkout -b feature/your-feature
3. Пушите изменения и создавайте PR.
  В PR обязательно:
  - Проверка линтерами.
  - Ревью кода.
4. После ревью мержите PR в main.
